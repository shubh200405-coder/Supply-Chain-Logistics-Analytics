# 📖 Project Overview

Supply chain and logistics organizations generate massive volumes of operational data every day. Transforming this data into actionable business insights is essential for improving operational efficiency, reducing transportation costs, optimizing fleet utilization, and supporting strategic decision-making.

This project demonstrates a complete end-to-end Business Intelligence workflow by integrating raw logistics datasets into PostgreSQL, performing SQL-based analysis, and developing interactive dashboards in Microsoft Power BI.

The project analyzes **more than 500,000 logistics records** distributed across **14 interconnected database tables**, covering customers, transportation routes, deliveries, fleet assets, drivers, maintenance records, fuel purchases, utilization metrics, and transportation safety.

The solution provides business users with an interactive dashboard to monitor operational performance, identify bottlenecks, improve resource utilization, and make data-driven decisions.

# 🗄 Database Structure

The PostgreSQL database consists of the following interconnected tables:

- Customers
- Drivers
- Trucks
- Trailers
- Facilities
- Routes
- Loads
- Trips
- Fuel Purchases
- Maintenance Records
- Delivery Events
- Safety Incidents
- Driver Monthly Metrics
- Truck Utilization Metrics

These tables were integrated into a centralized relational database to support analytical reporting and interactive business dashboards.
